# Technical Writing

Use this skill when writing docs, READMEs, guides, or specs.

## Principles
- Lead with the reader's goal; answer why before how.
- Use short sentences and active voice.
- Prefer concrete examples over abstract description.
- One idea per paragraph; use headings for scanability.
- Define terms on first use; avoid unexplained jargon.

## README structure
1. What it is and who it is for.
2. Quickstart / installation.
3. Usage examples.
4. Configuration and options.
5. Contributing and license.

## Workflow
1. Outline sections before drafting.
2. Draft, then cut every unnecessary word.
3. Test instructions by following them literally.
