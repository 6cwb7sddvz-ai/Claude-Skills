# Project Scaffolding

Use this skill when starting a new project or standardizing an existing one.

## Typical structure
- src/ for source code
- tests/ for tests
- docs/ for documentation
- README.md, LICENSE, and .gitignore at the root
- config files for linting, formatting, and CI

## Baseline tooling
- Linter and formatter (e.g., ruff/black, eslint/prettier)
- Test runner and coverage
- CI workflow for lint, test, and build
- Dependency and environment management

## Workflow
1. Confirm language, framework, and package manager.
2. Create the directory structure and root files.
3. Add linting, formatting, and test configs.
4. Add a CI workflow and a starter README.
